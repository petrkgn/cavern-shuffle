## История изменений

### 2026-05-11 — Рефакторинг отладочной системы: регистрируемые функции и клавиатурное управление

**Файлы:**
- `game/debug/debug_utils.lua` — заменены захардкоженные раскладки на систему `M.register_function()`, добавлена подробная документация архитектуры и инструкция по созданию новых отладочных кейсов
- `game/scripts/game.script` — добавлена обработка клавиш F7 (выбор функции) и F2 (запуск раскладки), заменён автозапуск на информационное сообщение
- `game/scripts/game_ui.gui_script` — добавлена защита от клавиатурных событий без координат x/y
- `input/game.input_binding` — добавлены биндинги `debug_select` (F7) и `debug_setup` (F2)

**Причина:** Старая система с захардкоженными раскладками и автозапуском не позволяла гибко тестировать разные игровые ситуации. Требовался расширяемый механизм для добавления новых отладочных кейсов без изменения основного кода игры.

**Результат:** Теперь при `DEBUG_MODE = true` игра запускается в обычном режиме и выводит подсказку. F7 циклически переключает дебаг-функции (с описанием в консоли), F2 создаёт выбранную раскладку. Игрок сам взаимодействует с картами — системы обрабатывают как обычно. Добавление нового кейса сводится к вызову `M.register_function()` в конце файла `debug_utils.lua`.

### 2026-05-02 — Валидация открытых карт подземелья после использования сокровищ

**Файлы:**
- `game/modules/systems/game_flow_system.lua` — добавлена функция `check_open_dungeon_duplicates` и её вызов после использования сокровищ для проверки дубликатов в открытых столбцах подземелья (исключая столбец с Минотавром).

**Причина:** При повторном использовании предмета сокровища (особенно полученного через джокера) карта в подземелье могла остаться на месте, создавая дубликат.

**Результат:** Теперь после использования сокровищ система автоматически проверяет верхние открытые карты в подземелье и удаляет дубликаты, если они обнаружены.

---

### 2026-04-14 — Создание единого отладочного модуля и функции для проверки задачи #2

**Файлы:**
- `game/debug/debug_utils.lua` — создан новый объединённый моду с `DEBUG_MODE`, `create_card_go()`, `setup_debug_deal()` и тремя старыми debug-функциями
- `game/modules/debug_utils.lua` — удалён (дубликат перенесён в `game/debug/`)
- `game/scripts/game.script` — добавлена интеграция debug-модуля через `DEBUG_MODE`
- `PLAN.md` — добавлен комментарий к задаче #2: логика подтверждения дубликата работает, требует проверки в боевой игре

**Причина:** Для проверки задачи #2 (дубликат под драконом) нужна специальная раскладка карт. Раньше debug-функции были в `game/modules/`, новый файл добавлен в `game/debug/` для лучшей организации.

**Результат:** При `DEBUG_MODE = true` игра запускается с готовой раскладкой: Treasure в inventory_1, Dragon+2_hearts в dungeon_3, A_hearts в party_1. Использование Treasure на A_hearts подтверждает что дубликат улетает на кладбище.

### 2026-04-05 — Прогрессбар отряда теперь корректно обновляется при использовании зелья силы

**Файлы:**
- `game/modules/systems/game_flow_system.lua` — добавлена отправка сообщения `potion_used` после установки баффа
- `game/scripts/game.script` — добавлен обработчик `potion_used`, `update_score_ui` теперь учитывает `temporary_power_buff.amount`

**Причина:** При использовании зелья силы прогрессбар не увеличивался, хотя бафф уже давал +5 к силе при проверке врагов и препятствий. Игрок не видел обратной связи.

**Результат:** Прогрессбар отряда теперь показывает `party_power + temporary_power_buff.amount`, давая игроку визуальную обратную связь при использовании зелья. Логика `can_defeat_enemy` и `can_clear_obstacle` не изменена — бафф учитывается как раньше.

### 2026-03-29 — Исправление эффекта частиц при трансформации джокера

**Файлы:**
- `game/modules/systems/game_flow_system.lua` — удалена функция `spawn_magic_poof()`, эффекты теперь создаются через `msg.post("transform_from", { spawn_poof = true })`
- `game/scripts/card.script` — эффект частиц создаётся в `update_flip()` при перевороте карты

**Причина:** При трансформации джокера в предмет не отображался эффект частиц. Старая реализация с `spawn_magic_poof()` была закомментирована и не использовалась.

**Результат:** Эффект частиц корректно отображается при трансформации Short Rest в джокера и при обратной трансформации джокера в предмет. Эффекты синхронизированы с анимацией переворота карты.

### 2026-03-29 — Восстановлена логика перемещения дракона при использовании Treasure

**Файлы:**
- `game/modules/systems/game_flow_system.lua` — добавлена функция `use_treasure`, отправляющая сообщение `adventurer_leveled_up`

**Причина:** После реворка Short Rest была утеряна функция `use_treasure`, которая отправляла сообщение о повышении уровня для перемещения дракона

**Результат:** Дракон корректно перемещается в следующую стопку после использования Treasure (из колоды и восстановленного из джокера)

### 2026-03-29 — Блокировка быстрых кликов по стопке explore во время анимации

**Файлы:**
- `game/modules/state.lua` — добавлен флаг `can_use` для стопки explore при инициализации и сбросе
- `game/modules/systems/input_system.lua` — проверка флага `can_use` перед обработкой клика
- `game/modules/systems/game_flow_system.lua` — установка `can_use = false` в начале анимации и сброс в колбэке

**Причина:** При быстрых кликах по стопке explore открывалось более 3 карт из-за отсутствия блокировки ввода во время анимации

**Результат:** Повторные клики игнорируются до завершения анимации добора карт

### 2026-03-15 — Исправление анимации перетасовки Short Rest

**Файлы:**
- `game/modules/systems/game_flow_system.lua` — заменён вызов `perform_shuffle_logic` на `use_short_rest`, удалена функция `perform_shuffle_logic`

**Причина:** После реворка Short Rest вызывалась упрощённая `perform_shuffle_logic` с одним проходом вместо трёх (Discard→Explore→Discard→Explore)

**Результат:** Карты снова летают 3 раза при использовании Short Rest, восстановлена полная анимация тасовки
