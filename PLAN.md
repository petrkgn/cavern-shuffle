# План разработки

## Инструкция

Этот файл содержит план разработки проекта Cavern Shuffle. Если в промте упоминается **план разработки**, необходимо обратиться к этому файлу для получения актуального списка задач.

**Важно:** Отмечать задачу выполненной (`[x]`) только после подтверждения от пользователя.

Задачи имеют сквозную нумерацию для удобной навигации. Для ссылки на задачу используйте формат `#<номер>` (например, «задача #1»).

---

## Задачи

### Bugs

1. - [x] **[BUG] Сломана логика отображения перетасовки карт после использования короткого отдыха**

   Должна быть анимация перемещения карт три раза, сейчас только один раз.

   **Файлы:** `game/modules/systems/game_flow_system.lua`, `game/scripts/card.script`

   **Причина:** Необходимо реализовать циклическую анимацию с тремя проходами карт через стопки.

2. - [ ] **[BUG] Карта закрыта драконом и используется сокровище при поднятии уровня**

   При использовании карты сокровищ вместо карты, которая закрыта драконом, дракон сдвигается, но карта под ним не улетает.

   **Файлы:** `game/modules/systems/rules_system.lua`, `game/modules/systems/render_system.lua`

   **Причина:** Не обрабатывается случай, когда под драконом находится другая карта.

   **Тест:** Создан debug-сценарий (`game/debug/debug_utils.lua`), логика подтверждения дубликата работает корректно (дубликат улетает на кладбище). **Требует проверки в боевой игре** при обычной раздаче.

3. - [ ] **[BUG] После хоровода при нажатии "начать снова" остаются плейсхолдеры в колонках подземелий**
  
   Нужно определить оптимальный момент для удаления плейсхолдеров: когда карты летят в хоровод или при нажатии кнопки "начать".
  
   **Файлы:** `game/modules/systems/game_flow_system.lua`, `game/modules/state.lua`
  
   **Причина:** Не очищается состояние подземелья при рестарте.

4. - [x] **[BUG] При клике по основной стопке блокировать клик до окончания анимации**

   Иначе артефакты отображаются некорректно (например, шесть карт вместо трёх).

   **Файлы:** `game/modules/systems/input_system.lua`, `game/scripts/game.script`

   **Причина:** Отсутствие флага блокировки ввода во время анимации.

5. - [ ] **[BUG] Предмет "Малая иллюзия" можно использовать на отряде** — **ТРЕБУЕТ ПРОВЕРКИ**

   ~~Это баг — предмет должен работать только с определёнными типами карт.~~

   **Статус:** Пользователь сообщил, что баг больше не воспроизводится. Предметы корректно работают только на своих стопках (включая предметы, созданные после использования джокера). Требуется дополнительная проверка при тестировании.

   **Файлы:** `game/modules/systems/rules_system.lua`, `game/modules/game_config.lua`

   **Причина:** Отсутствует проверка типа карты при использовании предмета.

6. - [x] **[BUG] При использовании карты сокровища на отряде дракон не переходит в следующую стопку**

   **Файлы:** `game/modules/systems/rules_system.lua`, `game/modules/systems/game_flow_system.lua`

   **Причина:** Не вызывается логика перемещения дракона после использования сокровища.

7. - [x] **[BUG] При использовании джокера на предмет нет эффекта частиц**

   **Файлы:** `game/modules/systems/game_flow_system.lua`, `game/scripts/card.script`

   **Причина:** Не вызывается `spawn_magic_poof` при трансформации джокера в предмет.

   **Решение:** Удалена функция `spawn_magic_poof()`, эффекты теперь создаются в `card.script` при перевороте карты через параметр `spawn_poof = true` в сообщении `transform_from`.

8. - [x] **[BUG] При использовании карты зелья силы на отряде не увеличивается прогрессбар отряда на 5**

9. - [x] **[BUG] При повторном использовании предмета сокровища, полученного с помощью джокера, карта в подземелье не улетает кладбище**                                      
                                                             
  **Файлы:** `game/modules/systems/rules_system.lua`, `game/modules/systems/game_flow_system.lua`                   

  **Причина:** Не корректно обрабатывается случай, когда карта сокровища была получена через джокера и затем использована повторно.  

### Features

10. - [ ] **[FEATURE] Halo-эффект на карте отряда с баффом зелья + подсказка туториала**

    На карту отряда, на которой применено зелье силы, нужно добавить визуальный halo-эффект, чтобы игрок не забывал о временном баффе. Также добавить подсказку туториала при клике на карту с halo.

    **Файлы:** `game/modules/systems/render_system.lua`, `game/scripts/card.script`, `game/modules/systems/tutorial_system.lua`, `game/modules/systems/tutorial_config.lua`

    **Причина:** Игрок забывает, что на карте активен бафф, и не знает какое препятствие можно пройти.

13. - [ ] **[FEATURE] Шейдер лавы на прогрессбарах**

   Добавить визуальный эффект текущей лавы на прогрессбары отряда или других игровых элементов. Шейдер должен создавать анимированный эффект раскалённой магмы с переливами.

   **Файлы:** `game/shaders/lava_progress.*`, `game/modules/systems/render_system.lua`

   **Причина:** Улучшение визуального восприятия прогресса и силы отряда, создание атмосферы подземелья.

14. - [ ] **[FEATURE] Эффект при перемещении дракона**

   При перемещении карты дракона — карта увеличивается, возможно дрожание экрана и искры пламени, звуковой эффект грохота или тяжёлой поступи. Создать впечатляющий визуальный и звуковой эффект, подчёркивающий масштабность события.

   **Файлы:** `game/modules/systems/game_flow_system.lua`, `game/scripts/card.script`, `game/modules/systems/render_system.lua`

   **Причина:** Перемещение дракона — важное игровое событие, требующее визуального и звукового подчёркивания.

15. - [ ] **[FEATURE] Вспышки при использовании предметов**

   При использовании предмета — вспышки с переливами по всему экрану (как в японских YouTube-роликах). При использовании короткого отдыха — визуальный эффект магической спирали ветра. Звуковые эффекты для каждого типа предмета.

   Референс: https://www.youtube.com/watch?v=KhR3aYID7Rc (2:31)

   **Файлы:** `game/modules/systems/game_flow_system.lua`, `game/scripts/card.script`, `game/modules/systems/render_system.lua`, `game/shaders/flash.*`

   **Причина:** Использование предметов должно сопровождаться впечатляющими эффектами для усиления ощущения от игрового процесса.

### Performance

11. - [ ] **[PERFORMANCE] Оптимизировать анимации ударов карт о дракона в хороводе**

   **Файлы:** `game/modules/systems/render_system.lua`, `game/scripts/card.script`

   **Причина:** Текущая реализация может вызывать просадки FPS при большом количестве карт.

12. - [ ] **[PERFORMANCE] В рендер-скрипте отписка и отправка только при смене разрешения**

   Проанализировать текущую реализацию слушателя на ресайз экрана на утечки. Оптимизировать обработку событий изменения размера окна — отписываться от слушателей и отправлять данные только при фактическом изменении разрешения.

   **Файлы:** `game/scripts/render.script`, `game/modules/systems/render_system.lua`

   **Причина:** Текущая реализация может создавать утечки памяти из-за отсутствия отписки от событий resize и избыточных вызовов.
